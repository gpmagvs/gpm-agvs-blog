## 注意事項

以下為規範書重點整理，供轉換架現場整合與除錯參考。完整流程與資料格式請以上方 PDF 為準。

### 背景與目的

- 轉換架作業需在 **Tray Barcode 辨識後同步上報顏色資訊**，供後續系統進行顏色追蹤與管控。
- 顏色來源依設備搭載狀況分為 **Color Sensor 自動感測** 或 **作業人員人工選取**。

### 顏色上報時機（Non-Handshake 模式）

| 情境 | 上報時機 |
|------|----------|
| 有 Color Sensor | Barcode Reader **成功讀取條碼後**，立即擷取 Color Sensor 色碼並上報 |
| 無 Color Sensor | Barcode Reader 拍照並完成條碼讀取後，**自動彈出顏色選擇畫面**，供人員手動選色並立即上報 |
| 有 Color Sensor 但讀取異常 | Color Sensor 為異常值或無法判讀時，**彈出顏色選擇畫面**，由使用者手動指定顏色並立即上報 |

:::info 即時上報
不論自動感測或人工選色，**所有顏色選擇皆須即時上報** 至 PLC。
:::

### 顏色選擇機制

- 人機介面須提供 **「顏色設定」按鈕**，供使用者在任意時機手動修改顏色。
- 按下按鈕後須跳出 **顏色選擇畫面** 供使用者挑選。
- 選定顏色後，系統應 **立即上報至 PLC**。

:::tip 人機介面建議
- 支援常用顏色快捷按鈕及 RGB / Hex 編輯模式。
- 選色後顯示確認結果（例如：顏色預覽 + 十六進位碼）。
:::

### 顏色上報格式與 PLC 位址

**資料格式**

- 顏色以 **Hexadecimal 色碼** 表示（格式 `#RRGGBB`）。
- 上報至 PLC 時，拆解為 R、G、B 三組 16 進位數字，以 **WORD** 方式寫入。

| 色碼範例 | RR（紅） | GG（綠） | BB（藍） |
|----------|----------|----------|----------|
| `#FF1200` | `0xFF` | `0x12` | `0x00` |

**PLC 記憶體對應**（連續三個 WORD）

| Address | 資料內容 | 說明 |
|---------|----------|------|
| W436 | RR | 紅色通道值 |
| W437 | GG | 綠色通道值 |
| W438 | BB | 藍色通道值 |

### 異常處理

| 情境 | 處理方式 |
|------|----------|
| Color Sensor 回傳值超出預期範圍或為 Null | 視為讀取異常，**主動跳出手動選色畫面** |
| 使用者未於設定時間內選擇顏色 | 系統應上報 **預設顏色（白色 `#FFFFFF`）** |
