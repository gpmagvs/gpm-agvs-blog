# 主畫面介紹

對應前端模組：`HomeView`、`TabContainer`（`StatusInfoCard`）、`AGVHeader`、`QuicklyActions`

## 功能概述

主畫面為車載 HMI 的預設操作介面，整合 AGV 即時狀態、異常紀錄、手動操作、設備交握與地圖顯示等功能。畫面分為 **頂部狀態列**、**左側操作區**、**中央分頁內容** 與 **底部資訊列** 四區，現場人員無需切換路由即可在同一畫面完成日常監控與操作。

## 進入方式

1. 車載觸控螢幕或瀏覽器開啟車載 UI 網址
2. 系統載入完成後自動進入主畫面（路由 `/`）
3. 若從其他頁面返回，可透過底部快捷列的 **Home** 圖示回到主畫面（Version2 介面）

### 中央分頁

主畫面中央以頁籤組織功能，預設顯示 **狀態** 分頁：

| 頁籤 | 說明 | 詳細文件 |
|------|------|----------|
| 狀態 | AGV 運行狀態、定位、載物、牙叉高度等 | 本節下方 |
| 異常紀錄 | 警報／警告歷史查詢與清除 | [異常紀錄](/docs/vehicle-guide/alarm-records) |
| 操作 | 手動移動、牙叉、IO 等控制 | [操作](/docs/vehicle-guide/operations) |
| 設備交握 (E84) | PIO／E84 交握訊號 | [設備交握](/docs/vehicle-guide/equipment-handshake) |
| 地圖顯示 | 地圖與本地任務 | [地圖顯示](/docs/vehicle-guide/map-display) |

登入 **God** 等高權限帳號後，可能額外出現 **CST Reader**、**Overview**、**Log** 等頁籤。

## 操作步驟

### 瀏覽主畫面各區域

1. 確認 [頂部狀態列](/docs/vehicle-guide/top-status-bar) 的 AGV 編號、使用者與連線是否正常
2. 查看左側 **Online Mode**／**Auto Mode** 是否符合預期作業模式
3. 在 **狀態** 分頁確認 SubState、定位狀態與載物 ID
4. 留意 [底部狀態列](/docs/vehicle-guide/bottom-status-bar) 的 CPU、RAM 與磁碟使用量

### 切換功能頁籤

1. 點選中央區域上方頁籤（如 **異常紀錄**、**操作**）
2. 各頁籤內容獨立；切換後左側操作區與頂部狀態列保持不變
3. 需要手動控制時，先確認已 [登入](/docs/vehicle-guide/user-login) 且為 **Offline**／**Manual**，再進入 **操作** 分頁

### 處理異常時的快速動線

1. 若畫面右下方出現 [即時警報顯示](/docs/vehicle-guide/realtime-alarm-display)，先閱讀異常碼與描述
2. 左側點選 **蜂鳴器關閉** 停止聲音（若需要）
3. 排除根因後點選 **異常復歸**，或至 **異常紀錄** 分頁查詢詳情
4. 定位或載物異常時，參考 **狀態** 分頁的 **定位狀態**、**Tag讀取** 與 **載物ID** 欄位

## 注意事項

- 主畫面為車載 UI 核心入口；多數日常操作可在此完成，無需離開本頁
- **OPERATOR** 表示尚未登入；進階操作前請先 [用戶登入](/docs/vehicle-guide/user-login)
- SubState 背景色會隨狀態改變（如 IDLE 為黃色）；**DOWN** 或 **ERROR** 需優先處理
- 派車系統連線異常（紅燈）時，部分按鈕可能被禁用，且無法正常接收派車任務
- 定位狀態為 **System Error** 或地圖比對率為 **0** 時，請勿進行自動導航，依 SOP 重新定位
- 底部磁碟使用率超過 70%／90% 時圖示會變色，請及時清理以避免影響車載系統運作
- 本截圖為 Fork AGV 範例；Inspection、Parts 等車型左側欄位與頁籤可能略有差異
