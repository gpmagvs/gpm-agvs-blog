# RESTful API 概覽

GPM 派車系統各服務提供 RESTful API。VMS 與 AGV 之間為**雙向 RESTful API** 通訊。

## 服務與 API 方向

| 方向 | 呼叫方 | 提供方 | 文件 |
|------|--------|--------|------|
| VMS → AGV | AGV 車載端 | VMS | [VMS 提供的 API](/docs/api/vms-agv) |
| AGV → VMS | VMS | AGV 車載端 | [AGV 提供的 API](/docs/api/agv-api) |

## Base URL

| 服務 | Base URL |
|------|----------|
| VMS | `http://{VMS_HOST}/api/AGV` |
| AGV 車載 | `http://{AGV_HOST}/api/AGV` |

實際 Host 與 Port 依現場部署設定。

## VMS 提供的 API（AGV 呼叫）

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/OnlineMode` | 查詢遠端模式 |
| POST | `/OnlineReq` | 請求上線 |
| POST | `/OfflineReq` | 請求下線 |
| POST | `/AGVStatus` | 上報運行狀態 |
| POST | `/TaskFeedback` | 上報任務進度 |
| POST | `/ReportMeasure` | 上報量測結果 |
| GET | `/CarrierVirtualID` | 取得 Virtual ID |
| POST | `/LeaveWorkStationRequest` | 請求離開工作站 |

詳見 [VMS 提供的 API](/docs/api/vms-agv)。

## AGV 提供的 API（VMS 呼叫）

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/RunningState` | 取得運行狀態 |
| GET | `/OnlineState` | 查詢 Online 狀態 |
| GET | `/agv_online` | 請求切換 ONLINE |
| GET | `/agv_offline` | 請求切換 OFFLINE |
| POST | `/Localization` | 定位 |
| POST | `/AGVSTrafficControllingNotify` | 交管通知 |
| GET | `/Notification` | 推送通知至 AGV 前端 |

詳見 [AGV 提供的 API](/docs/api/agv-api)。
