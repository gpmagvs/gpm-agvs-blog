# 實時警報

對應前端模組：警報頁（左側選單 **警報**，信封圖示）

## 功能概述

實時警報頁面集中顯示目前尚未清除的 **系統警報** 與 **設備警報**，供現場人員即時掌握 AGV、充電站、Zone 等異常，並執行清除警報、關閉蜂鳴器等操作。

## 進入方式

1. 登入 AGVS Web 介面
2. 點選左側選單 **警報**（若有未處理警報，圖示可能顯示紅色數字徽章）

## 畫面說明

![實時警報](/img/webui/realtime-alarm-page.png)

### 警報摘要列

頁面頂部顯示最新或最重要警報摘要，例如：

| 類型 | 範例 |
|------|------|
| 系統警報 | AGV 終點設備 Load Request Off |
| 設備警報 | 充電站 IO 模組連線異常 |

右側提供 **關閉蜂鳴器**、**清除警報** 等快捷按鈕。

### 系統警報

| 欄位 | 說明 |
|------|------|
| 時間 | 警報發生時間 |
| 設備名稱 | 相關 AGV 或系統物件（如 `AGV_001`） |
| 異常碼 | 警報代碼（如 1056） |
| 警報描述 | 中英文說明 |
| 異常類型 | 如 **ALARM**（紅色列）、**WARNING**（黃色列） |
| 任務名稱 | 關聯任務 ID（若有） |
| 發生地點 | Tag 或位置編號 |
| 持續時間(s) | 警報已持續秒數 |
| 清除警報人員 | 清除該警報的操作者 |

區塊右上角另有 **清除警報** 按鈕，可批次或區域清除系統警報。

### 設備警報

欄位與系統警報相同，來源通常為充電站、Zone、Port 等設備，例如：

| 等級 | 範例 |
|------|------|
| ALARM（紅） | `Charge_AOI_1` 充電站 IO 模組連線異常（7010） |
| WARNING（黃） | `Zone1` 料架 Port 有 ID 但無貨（2009） |

## 操作步驟

### 查看即時警報

1. 進入 **實時警報** 頁面
2. 先閱讀頂部 **警報摘要列**
3. 分別檢視 **系統警報** 與 **設備警報** 表格
4. 依 **異常類型** 色標區分 ALARM 與 WARNING 優先順序

### 關閉蜂鳴器

1. 現場確認告警原因已知晓或正在處理
2. 點選 **關閉蜂鳴器**（頂部摘要列或主畫面頂部告警列）
3. 蜂鳴器停止後，警報紀錄仍保留於列表中

### 清除警報

1. 依 SOP 完成異常排除或記錄
2. 在對應區塊點選 **清除警報**
3. 確認 **清除警報人員** 欄位更新；警報自即時列表移除
4. 歷史紀錄可至 [警報歷史查詢](/docs/guide/data-query/alarm-history-query) 追溯

### 關聯排查

1. 記下 **異常碼**、**設備名稱** 與 **任務名稱**
2. 若與任務相關，至 [任務列表](/docs/guide/basics/task-list) 或 [任務歷史查詢](/docs/guide/data-query/task-history-query) 查詢
3. 若與充電站相關，請依 SOP 檢查 IO 模組與連線

## 注意事項

- **清除警報** 不代表故障已排除，請先處理根因再清除
- ALARM 通常比 WARNING 優先處理；同一異常可能同時出現在主畫面頂部紅色告警列
- 左側選單警報圖示上的數字表示未讀或未處理警報數量
- 本頁為 **即時** 警報；歷史查詢請用 [警報歷史查詢](/docs/guide/data-query/alarm-history-query)
- 與 [告警處理](/docs/guide/maintenance/alarm-handling) 章節之維護流程搭配使用
