# 連線與模式切換


## 功能概述

頂部狀態列右側提供四組切換開關，用於控制 AGVS 整體 **操作模式**、**派工模式** 以及與上位 **HOST** 的連線與控制方式。切換前請確認現場 SOP 與許可權要求。

## 進入方式

1. 進入 [主畫面](/docs/guide/basics/home-page)
2. 檢視頂部狀態列右側的四組切換開關

:::info 服務狀態指示
切換區左側另顯示 **VMS**、**CIM**、**SECS Platform** 等服務執行狀態（綠色為正常）。服務異常時，模式切換可能無法生效或無法與 HOST 同步。
:::

## 畫面說明

![連線與模式切換](/img/webui/modes-change.png)

| 開關 | 標籤範例 | 說明 |
|------|----------|------|
| 操作模式 | **運轉**／**維護** | 控制 AGVS 是否處於正式運轉或維護作業狀態 |
| 派工模式 | **手動**／**自動** | 控制任務由人工派送或系統自動排程 |
| HOST 連線 | **Online**／**Offline** | 與上位 HOST／MCS 的連線狀態 |
| HOST 模式 | **Remote**／**Local** | 由 HOST 遠端控制或本地 AGVS 自行控制 |

### 操作模式

| 狀態 | 說明 |
|------|------|
| 運轉 | 正常生產模式，AGV 可接受派工與執行任務 |
| 維護 | 維護／工程模式，通常用於現場檢修、測試或排除異常 |

### 派工模式

| 狀態 | 說明 |
|------|------|
| 手動 | 任務需由操作員透過 **任務派送** 等方式手動建立 |
| 自動 | 由排程系統依規則自動產生並指派任務 |

### HOST 連線與模式

| 專案 | 說明 |
|------|------|
| HOST 連線 **Online** | AGVS 與上位系統連線，可接收 HOST 派工與狀態同步 |
| HOST 連線 **Offline** | 與上位斷線，AGVS 獨立運作 |
| HOST 模式 **Remote** | 派工與部分控制由 HOST 主導 |
| HOST 模式 **Local** | AGVS 本地控制，不依 HOST 遠端下指令 |

## 操作步驟

### 切換操作模式

1. 確認現場無進行中關鍵任務，或已依 SOP 暫停派工
2. 點選 **操作模式** 開關，在 **運轉** 與 **維護** 間切換
3. 切換後確認左側 [車輛狀態](/docs/guide/basics/vehicle-status) 與 [任務列表](/docs/guide/basics/task-list) 是否符合預期

### 切換派工模式

1. 點選 **派工模式** 開關
2. **手動** 模式下，使用 [主頁地圖](/docs/guide/basics/home-page-map) 的 **任務派送** 建立任務
3. **自動** 模式下，確認 HOST 或內部規則已啟用自動排程

### 切換 HOST 連線／模式

1. 依現場規範決定是否需先通知 HOST 或 MCS
2. 切換 **HOST 連線** 為 Online 或 Offline
3. 必要時再切換 **HOST 模式** 為 Remote 或 Local
4. 確認頂部 **CIM**、**SECS Platform** 指示與 [裝置狀態](/docs/guide/basics/equipment-status) 是否同步

## 注意事項

- **維護** 模式下請勿進行正式生產派工，除非 SOP 明確允許
- **HOST Offline** 或 **Local** 模式下，上位 MCS 可能無法下派任務或更新帳籍
- 切換模式前若頂部有紅色告警列，建議先處理告警或確認是否允許切換
- 派工模式與操作模式互相影響：例如維護模式下即使為自動派工，也可能不會執行新任務
- 模式切換通常需適當許可權；無許可權時請聯絡系統管理員
