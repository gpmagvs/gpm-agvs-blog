# 任務列表


## 功能概述

任務列表面板用於監控派車任務的執行情況。分為 **未完成任務** 與 **已完成任務** 兩個子分頁，可分別檢視進行中／待執行任務，以及歷史完成紀錄。

## 進入方式

1. 進入 [主畫面](/docs/guide/basics/home-page)
2. 確認左側面板已展開
3. 點選 **任務列表** 分頁
4. 切換 **未完成任務** 或 **已完成任務** 子分頁

---

## 未完成任務

顯示目前 **執行中** 與 **等待執行** 的任務，卡片會即時更新進度與狀態訊息。

![未完成任務](/img/webui/miss-status-incompleted.png)

### 任務分組

| 分組 | 說明 |
|------|------|
| 執行中 | 已指派 AGV、正在執行的任務 |
| 等待執行 | 已建立但尚未指派車輛，或等待排程的任務 |

### 任務卡片欄位

| 欄位 | 說明 |
|------|------|
| 動作 | 任務型別，如 **搬運**、**充電** 等 |
| 任務 ID | 系統產生的唯一識別碼（如 `Local_20260615_165551822`） |
| 進度步驟 | 搬運任務通常為：移站 → 取貨 → 移站 → 放貨；已完成步驟顯示綠色勾選 |
| 起點／終點 | 任務來源與目的站點（Tag）及樓層 |
| 接收時間 | 任務建立時間 |
| 派工人 | 建立或觸發任務的使用者 |
| 車輛 | 執行 AGV 編號；等待執行時可能顯示 **Not Assigned** |
| 載具 ID | 搬運任務所對應的 Carrier ID |
| 狀態訊息 | 目前派工或路徑規劃的詳細說明（紅色文字為提示或異常訊息） |

### 操作按鈕

| 按鈕 | 適用 | 說明 |
|------|------|------|
| 中止任務 | 執行中 | 強制中止正在執行的任務 |
| 取消任務 | 等待執行 | 取消尚未開始的任務 |

### 操作步驟

#### 檢視執行中任務進度

1. 切換至 **未完成任務** 分頁
2. 在 **執行中** 區塊找到目標任務卡片
3. 依進度步驟確認目前階段（移站、取貨、放貨等）
4. 可對照中央地圖上 AGV 位置與路徑

#### 中止或取消任務

1. 執行中任務：點選 **中止任務**，確認後 AGV 停止執行
2. 等待執行任務：點選 **取消任務**，將任務自佇列移除
3. 中止／取消後，請確認地圖上 AGV 與站點狀態是否恢復正常

## 已完成任務

以表格列出已結束的任務紀錄，供查詢與追溯。

![已完成任務](/img/webui/miss-status-completed.png)

### 表格欄位

| 欄位 | 說明 |
|------|------|
| 任務 ID | 任務唯一識別碼 |
| 執行車輛 | 完成任務的 AGV 編號 |
| 接收時間 | 任務建立時間 |
| 結束時間 | 任務完成時間 |
| 狀態 | 固定為 **完成**（綠色標籤） |
| 動作 | 任務型別，如 **充電**、**搬運** |
| 貨物 ID | 搬運任務的 Carrier ID；充電任務可能為空 |
| 重新執行 | 以相同條件重新觸發任務 |

### 任務摘要提示

滑鼠移至任務列上方時，可顯示摘要資訊：

- 起點 Tag
- 終點 Tag
- 任務總耗時

### 操作步驟

#### 查詢歷史任務

1. 切換至 **已完成任務** 分頁
2. 依接收時間、執行車輛或動作型別瀏覽表格
3. 將遊標移至任務列，檢視起訖點與耗時摘要

#### 重新執行任務

1. 在已完成列表中找到目標任務
2. 點選 **重新執行**
3. 確認派工引數後送出；新任務將出現在 **未完成任務** 分頁

---

## 注意事項

- **中止任務** 可能使 AGV 停在非預期位置，請依 SOP 確認現場安全後再操作
- 等待執行且 **Not Assigned** 的任務，表示排程尚未指派車輛，若長時間未派工請檢查 VMS 與車隊狀態
- 已完成任務的 **重新執行** 會建立新任務，不會修改原紀錄
- 搬運任務進行中，地圖上終點站點可能顯示 **(Reserved)** 保留狀態，表示已被該任務佔用
- 若任務卡片出現異常狀態訊息，請先記錄任務 ID 與 Carrier ID，再聯絡工程人員排查
