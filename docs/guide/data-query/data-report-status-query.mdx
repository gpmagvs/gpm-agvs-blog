# 數據上報狀態查詢

對應前端模組：數據上報狀態（左側選單 **資料查詢** · 數據上報）

## 功能概述

數據上報狀態查詢以月曆方式顯示各日向上位系統（如 CIM／SECS）上報歷史資料的結果，包含任務歷史、警報歷史、車輛稼動率與磁碟使用狀況等項目，並可對失敗日期執行 **重新上報**。

## 進入方式

1. 登入 AGVS Web 介面
2. 點選左側選單 **資料查詢**
3. 進入 **數據上報狀態查詢** 頁面

## 畫面說明

![數據上報狀態查詢](/img/webui/data-report-history-query.png)

### 月曆控制

| 項目 | 說明 |
|------|------|
| 月份選擇 | 選擇要檢視的年月（如 2026 年 6 月） |
| 上一年／上一月 | 切換至前一年或前一月 |
| 今天 | 跳至含今日之月份並標示當日 |
| 下一月／下一年 | 切換至後一月或後一年 |

### 日期儲格狀態

每個日期儲格可能顯示下列上報項目（紅字通常表示 **未上報** 或失敗）：

| 項目 | 說明 |
|------|------|
| 任務歷史 | 任務歷史資料上報狀態 |
| 警報歷史 | 警報歷史資料上報狀態 |
| 車輛稼動率 | 稼動率統計上報狀態 |
| 磁碟使用狀況 | 磁碟使用量資料上報狀態 |

| 儲格狀態 | 說明 |
|----------|------|
| 未上報（各項目紅字） | 該日部分或全部項目尚未成功上報 |
| 重新上報 | 藍色按鈕，觸發該日資料重新上報 |
| 尚未進行 | 當日尚未執行上報，或未來日期（灰色） |
| 當日高亮 | 淺藍底色通常表示系統當天 |

## 操作步驟

### 檢視上報狀態

1. 選擇目標 **月份**
2. 逐日查看儲格內四項上報狀態
3. 找出標示 **未上報** 的日期與項目

### 重新上報

1. 在失敗日期儲格點選 **重新上報**
2. 等待系統處理完成
3. 重新整理或切換月份後確認狀態是否已更新

### 對照本地資料

1. 若 **任務歷史** 未上報，可至 [任務歷史查詢](/docs/guide/data-query/task-history-query) 確認本地是否有資料
2. 若 **警報歷史** 未上報，可至 [警報歷史查詢](/docs/guide/data-query/alarm-history-query) 對照
3. 若 **車輛稼動率** 未上報，可至 [稼動率查詢](/docs/guide/data-query/availability-query) 確認統計是否完整

## 注意事項

- 上報成功與否受 **CIM**、**SECS Platform** 連線影響；頂部指示異常時可能無法上報
- **HOST Offline** 或 **Local** 模式下，上位可能無法接收上報
- **尚未進行** 表示當日排程尚未執行或未來日期，不代表失敗
- **重新上報** 可能產生重複資料，請依 HOST／MCS 規範操作
- 磁碟使用狀況未上報時，請一併確認伺服器儲存空間（底部狀態列）
